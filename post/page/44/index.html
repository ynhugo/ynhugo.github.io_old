linux环境搭建hugo博客

搭建hugo博客 这里跟着官网教程进行配置，刚开始折腾hugo博客的话，最重要的是折腾成功，官网教程简单实用。 安装hugo apt install hugo 创建站点目录 然后在硬盘中找一个自己想要存放博客的位置，创建站点目录，例如我要创建的站点目录的名字叫做hugo1. hugo new site hugo1 然后名字叫做hugo1的站点目录就创建好了。 配置博客主题 还是刚开始说的，能够快速折腾成功还是挺重要的，因此还是根据官方教程操作，配置一个主题名叫做ananke的主题。 git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke themes/ananke 然后在站点目录的hugo.toml文件中最下边添加一行内容，添加的内容是 theme = 'ananke' 可以用用shell命令添加 echo "theme = 'ananke'" >> hugo.toml 创建博客 创建属于该博客的第一篇内容，名字就叫"我的第一篇博客"吧 hugo new posts/我的第一篇博客.md 然后在站点目录下的content/posts/下就相应的出现了一个文件"我的第一篇博客.md" 本地预览 hugo server -D 部署博客 配置github仓库 在github创建账户，然后创建仓库，仓库命名格式为"yourname.github.io"，配置仓库的主分支为master，配置仓库设置界面的pages的主分支为master。 在本地生成ssh密钥 ssh-keygen 三次回车后密钥就生成了，生成的密钥在家目录下的.ssh目录下，将.ssh/的id_rsa.pub的内容复制到剪切板，然后 回到github网站，在帐户设置的SSH and GPG keys界面中添加剪切板的ssh公钥，然后给添加该密钥的记录随便起一 个名字用于区分可以了。 配置hugo 将自己仓库的名字yourname.github.io添加到站点目录的hugo.toml配置文件中，将示例链接替换掉，然后设置博客 的语言，接着设置博客的名称，最后一行也就是第四行是咱刚开始就添加过了的，所以不用修改。 baseURL = 'yourname.github.io' languageCode = 'zh-cn' title = 'MyBlogs' theme = 'ananke' 部署 先将博客的草稿模式关闭，将true修改为false，然后回到站点目录的第一级目录，再进入publib目录，先添加github的仓库链接，然后执行git命令进行部署hugo的操作。 git remote add origin git@github.com:yourname/yourname.github.io.git git add . git commit -m "upload" git push -u origin master 部署失败解决方法 如果部署过程出现问题，可以执行以下命令来解决 ...

termux环境搭建alist教程

搭建alist教程 alist配置官方文档。 安装alist 安装程序后在命令行界面运行就可以了，第一次运行的时候先获取到用户名和密码，然后根据提供的用户名和密码进行网页访问并修改密码。 后台运行 在termux的家目录编辑".bashrc"文件，如果没有这个文件可以创建这个文件，该命名的文件在启动termux后可以自动运行，不过alist不能重复启动，为避免使用ssh连接termux时再次启动启动alist，我写了一个脚本进行判断，如果alist已经运行就不再重复运行，执行命令时产生的附带文件如nohup.out和null也通过判断是否存在，如果存在则删除，在这一部分我在网上找了两个链接参考了一下，分别是shell判断进程是否存在，不存在则启动和Shell脚本实用小技巧-教你屏蔽执行命令的所有显示信息，包含错误信息和Shell 判断文件或文件夹是否存在（不存在则创建），这样这个脚本就可以实现alist在后台运行了，并且不需要自己手动做任何操作了。 想要完成上面描述的效果，则需要将下面的脚本内容添加到".bashrc"中 #!/data/data/com.termux/files/usr/bin/bash ps -ef | grep "alist" | grep -v "grep" > /dev/null 2>&1 if [ $? -ne 0 ] then nohup alist server & fi if [ -e "/data/data/com.termux/files/home/null" ] then rm /data/data/com.termux/files/home/null fi if [ -e "/data/data/com.termux/files/home/nohup.out" ] then rm /data/data/com.termux/files/home/nohup.out fi[<35;177;4m 最后启动该文件即可 source .bashrc alist操作方式 使用alist的命令有两个就够用了，分别是用于获取alist密码的运行alist服务的。 获取用户名和密码 alist admin 运行alist服务 alist server 运行服务之后通过提示的端口进行网页访问，链接格式是这样的"127.0.0.1:5244"，通过这个链接进行访问你就可以访问alist的界面了。 添加存储 接下来就需要添加存储了，将百度网盘或者阿里云网盘添加进alist存储，也可以添加本地目录； 第一次访问alist时界面什么都没有，未来添加过网盘后首页应付出现相应的网盘了，添加网盘首先要做的操作是点 击首页最底部的管理，然后再点击存储，接着点击添加，接下来就可以选择要添加的网盘的类型了，添加好的网盘可以在主页浏览。 浏览官方文档 官方文档 刚开始接触alist只需要看"安装"和"添加存储"就够用了，相关内容介绍的也都比较到位，至少我们平时用的百度网 盘和阿里云网盘是没有问题的，其他网盘有待折腾，包括网上也会有许多不错的教程可以参考学习。 添加百度网盘 需要关注的有两点，刷新令牌和web代理； 刷新令牌可以根据官方文档进行操作获取并进行添加，web代理也勾选一下，挂载路径随便设置，格式是一个斜杠加 自定义名称，就像这样"/BaiduNetdisk"，剩下的不需要动，接着点击保存就可以了。 ...

vmware安装android系统教程

参考链接：在VMware上体验了一把Android x86 9 下载安卓9.0映像文件 我是在这个网站下载的https://www.fosshub.com/Android-x86.html，选择的是Android-x86 64-bit ISO file，对应版本是9.0-r2-k49。 打开vmware进行安装 1.点击文件，接着点击创建虚拟机； 2.选择自定义，然后点击下一步； 3.硬件兼容性默认，然后点击下一步； 4.点击稍后安装操作系统，然后点击下一步； 5.客户机系统选择Linux，版本选择其他Linux 5.x 内核64位，然后点击下一步； 6.虚拟机名称自定义，选择安装位置，然后点击下一步； 7.内核数量根据自己电脑硬件情况自定义，然后点击下一步； 8.内存选择2GB或者4GB，然后点击下一步； 9.网络类型选择使用网络地址转换(NAT)，然后点击下一步； 10.IO控制器类型默认，然后点击下一步； 11.磁盘类型默认，然后点击下一步； 12.创建新虚拟磁盘，然后点击下一步； 13.最大磁盘大小这里我分配了40GB，可以自由分配，并且我选择了将虚拟磁盘拆分成多个文件，然后点击下一步； 14.在指定磁盘文件这里，我的做法是在安卓系统的安装位置中再建一个用来存放磁盘文件的子目录，作为磁盘文 件的目录，然后点击下一步； 15.在这一步中，先点击自定义硬件，然后点击新CD/DVD(IDE)，在右侧点击使用ISO映像文件，接着点击浏览到下载的安卓映像文件所在的目录选择安卓映像文件，然后点击打开，点击关闭，最后点击完成; 16.为保险起见，我为安卓系统做一个快照，在安卓系统的标签鼠标右键点击，在弹出的菜单中点击快照，点击拍摄快照，快照的名称和描述可以自定义，再次点击拍摄快照，一秒到两秒钟的时间快照就制作完成了，未开 机状态下的快照制作过程花费时间是最短的； 17.点击开启此虚拟机，底部弹窗可以叉掉，点进安卓系统的界面的时候鼠标消失了，可以通过快捷键ctrl+alt 恢复鼠标，不过目前要安装安卓系统，因此先不恢复鼠标，在安卓系统的启动界面通过上下方向键选择第三个Installation - Install Android-x86 to harddisk选项后敲回车，加载后进入选择分区界面，用上下方向键选择第一个Create/Modify partitions后，默认选择OK，然后敲回车，再次弹出窗口，默认选择No，再次敲回车，进入创建分区界面，将光标通过左右方向键移到New后敲回车，选择primary后敲回车，在设置分区空间大小界面什么都不要做，再次敲回车，在分区界面默认选中了sda1，将移动光标选中write，然后敲回车，接着分区界面提示yes or no，输入yes，然后敲回车，加载之后选中Quit，然后敲回车，在Choose filesystem界面中默认选中了sda1，光标默认选中OK，其他什么都不做，直接敲回车，在下一个界面中选ext4，然后敲回车，在Confirm界 面选中Yes，然后敲回车，加载后再次在Confirm界面选中Yes，然后敲回车，接下来进入到了Question界面 ，依然是选中Yes，然后敲回车开始了安卓系统安装的过程，加载后先什么都不要做，鼠标右键点击当前安卓系统 的标签，在可移动设备的列表中把CD/DVD(IDE)的连接断开，在进行下一步操作之前如果想保险一点可以先将安 卓系统关机并做一个快照然后重新启动，系统在加载后进入到了GRUB启动模式选择界面，选择第二个模式Android-x86 9.0-r2（Debug mode），然后敲回车，系统开始加载，耐心等待一会儿后，尝试敲回车，如果没有反应就继续加载过程，然后再次尝试敲回车，出现一行只有:/android #，并且旁边有光标在来回闪烁，意味着进入到了安卓系 统的终端环境中，接下来要做几个操作，在命令行界面输入mount -o remount,rw /mnt后回车，执行该命令后在下一行返回一行字符串，不用管，输入下一条命令vi /mnt/grub/ment.lst后敲回车，修改ment.lst文件，进入这 个文件后点击键盘的字母i进入编辑模式，然后将光标通过方向键移动到需要修改的位置在这个文件中有四个title，分别是Android-x86 9.0-r2、Android-x86 9.0-r2（Debug mode）、Android-x86 9.0-r2（Debug nomodeset）和Android-x86 9.0-r2（Debug video=LVDS-1 :d），需要修改语句是的是第一个titleAndroid-x86 9.0-r2下边的kernel /android-9.0-r2/kernel quiet root=/dev/ram0 SRC=/android-9.0-r2，将该条语句中的root 改为nomodeset_root，然后点击按键esc退出编辑模式，再执行快捷键shift+;，这个快捷键由按键shift和 标点符号;组成，执行快捷键后进入底行模式，输入字母wq后回车便退出该文件，接下来在系统标签右击展开 菜单，点击电源列表的重新启动客户机，在模式选择界面选中第一个模式后回车，或者你也可以什么都不做，系统会自动默认选择第一个模式并运行，耐心等待一会儿进入安卓系统。 配置安卓系统 第一次启动进行安卓系统开机时需要设置语言，把语言设置成简体中文，然后点击开始，跳过网络设置，一些隐私设置能不勾选就不要勾选，屏幕锁定也不设置，点击以后再说，主屏幕应用选择Quickstep并设置始终选择，至此 安卓系统就配置完成了。
